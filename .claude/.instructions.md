# コーディング規約

Claude Code がコードを生成・編集する際に従うべきルールを定義する。

## 名前空間（namespace）規則

- 命名規則: `Technopro.Othello.<レイヤー>.<サブレイヤー>` の形式に統一する
- 対応例:

| 旧 namespace | 新 namespace |
|---|---|
| `Othello.Core.Models` | `Technopro.Othello.Core.Models` |
| `Othello.Core.Rules` | `Technopro.Othello.Core.Rules` |
| `Othello.Core.Game` | `Technopro.Othello.Core.Game` |
| `Technopro.Othello.Core.AI` | `Technopro.Othello.Core.AI` |
| `Othello.WPF.ViewModels` | `Technopro.Othello.WPF.ViewModels` |
| `Othello.WPF.Converters` | `Technopro.Othello.WPF.Converters` |
| `Othello.Tests.*` | `Technopro.Othello.Tests.*` |

- 各 .csproj には `<RootNamespace>Technopro.Othello.XXX</RootNamespace>` を設定する
- XAML の `x:Class` と `clr-namespace:` も同様に更新する

## 実装パターン

### async Task + CancellationToken パターン
`GameViewModel` の非同期メソッド（`ProcessAIMoveAsync`, `CheckAndProcessNextTurnAsync` 等）を追加・修正する際は以下のパターンを踏襲する。
新規ゲーム開始時に `_cts.Cancel()` が呼ばれ、`OperationCanceledException` で正常終了する。

```csharp
private async Task SomeAsyncMethodAsync(CancellationToken ct)
{
    try
    {
        await Task.Delay(..., ct);          // キャンセルで OperationCanceledException
        ct.ThrowIfCancellationRequested();  // 長時間処理の後に明示チェック
        // ... 処理
    }
    catch (OperationCanceledException)
    {
        // 新規ゲーム開始による正常なキャンセル。状態は StartNewGame が管理する。
    }
    catch (Exception ex)
    {
        StatusMessage = $"エラー: {ex.Message}";
    }
    finally
    {
        if (!ct.IsCancellationRequested) IsAIThinking = false;
    }
}
```

呼び出し側（fire-and-forget）:
```csharp
_ = SomeAsyncMethodAsync(_cts!.Token);
```

### 拡張メソッド
列挙型に機能を追加するときは、同じファイル内の `XxxExtensions` static クラスに拡張メソッドとして定義する（例: `PlayerColorExtensions`, `GameStateExtensions`, `DifficultyLevelExtensions`）。

### MoveResult の生成
`MoveResult` は直接コンストラクタを呼ばず、ファクトリメソッドを使う。

```csharp
MoveResult.Success("メッセージ", flippedPieces); // 成功
MoveResult.Failure("エラー理由");                 // 失敗
```

## C# コメント規約

- **クラス・インターフェース・列挙型**: XML ドキュメントコメント `/// <summary>` で概要を記述する
- **メソッド・プロパティ**: `/// <summary>` による概要に加え、`/// <param name="...">` と `/// <returns>` で引数・戻り値を記述する
- **条件分岐（if / switch 等）**: 条件の意味・判定理由をインラインコメント `//` で記述する
- **重要な処理**: アルゴリズムの意図・副作用・注意点をインラインコメントで記述する

```csharp
/// <summary>ボードの指定位置へ石を置き、結果を返す。</summary>
/// <param name="position">配置先の座標（0-7, 0-7）</param>
/// <returns>成功/失敗と反転した石のリストを含む結果</returns>
public MoveResult MakeMove(Position position) { ... }
```

## Python コメント規約

- **クラス・関数**: docstring（`"""..."""`）で概要・引数・戻り値を記述する
- **条件分岐**: 条件の意味をインラインコメント `#` で記述する
- **重要な処理**: アルゴリズムの意図をインラインコメントで記述する

```python
def get_flips(board, r, c, player):
    """指定位置に stone を置いた際に反転される相手石の座標リストを返す。"""
```
